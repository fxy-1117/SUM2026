# ICTAI 2026 Enthymeme Evaluation

This repository contains the cleaned evaluation pipeline, fixed data, and final
result files used for the enthymeme experiments.

## Layout

```text
data/
  anli/
    original/
      test.jsonl
      test-labels.lst
    generated/
      one.csv
      two.csv
      three.csv
  arct/
    original/
      test.tsv
    generated/
      one.csv
      two.csv
      three.csv
  data_report.csv

results/
  sweep.csv
  metrics.csv
  best_f1.csv
  best_f1.tex
  entailment.csv
  entailment.tex
  parameter_sweep_figure.tex
```

Local caches, debugging scripts, and non-final result files are not required
for reproducing the submitted tables and are excluded from the release.

## Environment

Use the same conda environment used during the experiments:

```powershell
conda activate eesnli
```

If dependencies need to be installed manually, start from `requirements.txt`.

## Run The Experiments

Run commands from the repository root. The scripts use seed `1129` and write to
`results/`.

The main parameter sweep uses `FIX_NUMBER = 150` and writes the metrics needed
by the Best F1 table.

```powershell
python evaluate.py
```

Outputs:

```text
results/sweep.csv
results/metrics.csv
```

Extract the Best F1 table from the sweep metrics:

```powershell
python make_best_f1.py
```

Outputs:

```text
results/best_f1.csv
results/best_f1.tex
```

Run the entailment-only table with `FIX_NUMBER = 250`:

```powershell
python evaluate_entailment.py
```

Output:

```text
results/entailment.csv
results/entailment.tex
```

The committed result files were generated with these commands:

```powershell
python evaluate.py
python make_best_f1.py
python evaluate_entailment.py
```

## Data Policy

The final data are fixed, submission-ready subsets. The evaluation scripts do
not call an LLM API; generated chains are already included in `data/`.

For ANLI, the generated files contain the selected one-, two-, and three-step
chains. For ARCT, the one-step file includes the selected unique keys plus
high-difference duplicate-key rows used to fill the final fixed-size sweep; the
two- and three-step files use the selected unique-key rows.

The evaluation scripts use seed `1129`. Shuffling still matters because the
pipeline skips invalid rows, rows whose prover result is `both`, and rows that
do not match the class-specific evaluation condition. The fixed data files make
the pool reproducible, while the seed fixes the order in which valid rows are
counted.

For the parameter sweep, ARCT `original` deduplicates exact `Premise + Claim`
pairs. ARCT one-step keeps the final top-up rows in
`data/arct/generated/one.csv`, so it can reach the target count more reliably.
For the entailment-only table, all ARCT step types deduplicate exact
`Premise + Claim` pairs, matching the final unique-key setup used for that
comparison.

Logic cache files are generated by dataset under `.cache_eval/logic_sentences/`.
Neural similarity/NLI caches are generated by dataset under
`.cache_eval/neural_cache/`. These caches speed up repeated runs but are not
required for correctness and are not committed.
